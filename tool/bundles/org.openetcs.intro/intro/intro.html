Connecting the Model Repository

This assumes that you installed openETCS and opened it.

You have two options here: You can (1) use an existing clone of the model repository, if you cloned it before. Or you can (2) tell Eclipse to clone the repository for you.

(1) use an existing clone of the model repository

  • File | Import ...
  • Git | Projects from Git
  • Existing local repository
  • Add...
  • Browse to the local copy of the modeling repository and hit Finish
  • Click Next (Import existing projects)
  • Click Next again: Deselect all except "model"
  • Click finish: You are done.

(2) tell Eclipse to clone the repository for you

  • File | Import ...
  • Git | Projects from Git
  • Clone URI
  • Provide this URI: git@github.com:openETCS/modeling.git
  • Click Next: You can clone all branches (recommended) or just "master"
  • Click Next
  • Click Next again (Import existing projects)
  • Click Next again: Deselect all except "model"
  • Click finish: You are done.

Next step

No matter whether you followed (1) or (2), there should be one Project in the workspace (called "model") with the subfolders sysml and subset26.

Read-only or writable?

If you configured SSH and are a committer to the project, then you will be able to push data back to the server. Files will always be writable, no matter whether you have push privileges or not.
